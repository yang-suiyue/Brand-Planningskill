# 项目主档字段定义

## 核心概念

项目主档（Project Master）是品牌策划全过程的**唯一信息源**，所有后续章节内容都必须从主档衍生。主档采用状态标注系统，确保信息完整性和可追溯性。

## 字段分类

### 必填字段（✅已确认）

必须在建立项目时收集，这些是生成策划案的基础信息：

| 字段名 | 说明 | 示例 |
|--------|------|------|
| 品牌名称 | 品牌正式名称 | 「元气森林」 |
| 所属赛道 | 品牌所在赛道 | ToC新消费 |
| 赛事级别 | 参赛级别 | 省赛/国赛 |
| 核心痛点 | 品牌当前面临的核心问题 | 年轻消费者认知度不足 |
| 目标用户 | 品牌的核心目标客群 | 18-30岁年轻女性 |

### 重要字段（⚠️待补充）

建立项目后可逐步补充，是策划案的重要组成部分：

| 字段名 | 说明 | 示例 |
|--------|------|------|
| 品牌所属企业 | 品牌所属公司主体 | 元气森林（北京）食品科技集团有限公司 |
| 核心产品/服务 | 主要产品线或服务 | 无糖气泡水、功能性饮料 |
| 商业模式 | 盈利方式 | B2C直销+渠道分销 |
| 现有品牌资产 | 已有品牌元素 | 品牌定位、品牌口号、视觉体系 |

### 可选字段（🔍待验证）

需要通过Research阶段补充的信息：

| 字段名 | 说明 | 数据来源 |
|--------|------|----------|
| 市场规模数据 | 行业市场规模 | 市场调研报告 |
| 竞品信息 | 主要竞争对手分析 | 公开资料 |
| 财务数据 | 营收、利润等 | 企业年报/估计 |
| 政策环境 | 行业相关政策 | 政府文件 |

## 状态标注规则

- ✅ **已确认**：用户已提供且核实无误的信息
- ⚠️ **待补充**：用户已知但尚未完整提供的信息
- 🔍 **待验证**：需要通过Research补充的公开信息

```markdown

- 所属赛道：__________

- 核心痛点：__________
- 目标用户：__________

## 品牌信息（⚠️待补充）
- 品牌所属企业：__________
- 核心产品/服务：__________

- 现有品牌资产：__________

## 调研信息（🔍待验证）
- 市场规模数据：__________
- 竞品信息：__________
- 财务数据：__________

```

## 使用规则

1. **唯一信息源**：所有章节生成必须引用主档信息
2. **状态追踪**：定期更新字段状态
3. **信息溯源**：任何数据必须标注来源
4. **版本管理**：重大信息变更需记录版本历史
